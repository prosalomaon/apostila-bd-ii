Aula 31: Introdução ao NoSQL

Explorando modelos de dados não estruturados, escalabilidade e o Teorema CAP.

Teoria & Conceitos

O termo NoSQL (Not Only SQL) refere-se a bancos de dados que não utilizam o modelo relacional tradicional. Eles surgiram para lidar com o grande volume, velocidade e variedade de dados (Big Data).

  • Flexibilidade de Esquema: Não exige uma estrutura rígida de tabelas e colunas.
  • Escalonamento Horizontal: Projetados para rodar em clusters de servidores comuns.
  • Teorema CAP: Um sistema distribuído só pode garantir dois dos três pilares: Consistência, Disponibilidade e Tolerância a Partição.

Tipos de NoSQL

1. Documentos (ex: MongoDB, CouchDB)
2. Chave-Valor (ex: Redis, DynamoDB)
3. Colunas (ex: Cassandra, HBase)
4. Grafos (ex: Neo4j)

Dicas & Lembretes

NoSQL não substitui o SQL. NoSQL é uma ferramenta específica para casos de uso onde a estrutura dos dados muda muito ou onde a performance de leitura/escrita em massa é crítica.

Como aplicar na Prática

Use bancos de documentos para armazenar perfis de usuários em redes sociais, onde cada usuário pode ter informações diferentes (uns têm Instagram, outros não, outros têm 10 endereços, etc.).
