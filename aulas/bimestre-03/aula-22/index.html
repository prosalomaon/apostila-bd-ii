Aula 22: Stored Procedures

Automatizando tarefas e encapsulando lógica de negócio diretamente no servidor de Banco de Dados.

Teoria & Conceitos

Uma Stored Procedure (Procedimento Armazenado) é um conjunto de comandos SQL compilados e armazenados no servidor. Elas permitem:

  • Reutilização: Escreva uma vez, execute em qualquer lugar do sistema.
  • Segurança: Usuários podem executar a procedure sem ter acesso direto às tabelas subjacentes.
  • Performance: Reduz o tráfego de rede e aproveita o plano de execução em cache.

Exemplo de Código

Criação de uma procedure para registrar uma venda e atualizar o estoque:

CREATE PROCEDURE RegistrarVenda
    @ProdutoID INT,
    @Quantidade INT
AS
BEGIN
    BEGIN TRANSACTION
        INSERT INTO Vendas (ProdutoID, Quantidade, DataVenda)
        VALUES (@ProdutoID, @Quantidade, GETDATE());

        UPDATE Produtos 
        SET Estoque = Estoque - @Quantidade
        WHERE ProdutoID = @ProdutoID;
    COMMIT TRANSACTION
END;

Dicas & Lembretes

Sempre use BEGIN TRANSACTION e COMMIT/ROLLBACK dentro de procedures que alteram dados, garantindo a atomicidade da operação.

Como aplicar na Prática

Procedures são ideais para processos complexos de fechamento de mês, geração de relatórios consolidados e qualquer operação que envolva múltiplas tabelas e regras de validação.
