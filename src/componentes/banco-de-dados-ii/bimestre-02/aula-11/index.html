--- title: "DQL Avançada - Predicados IN, LIKE e BETWEEN" aula_numero: "11" description: "Explorando os predicados lógicos LIKE, BETWEEN e IN para filtragem avançada de dados." tags: - bimestre-02 content_teoria: |

Os predicados são filtros lógicos que determinam a "sobrevivência" de um registro no conjunto de resultados, funcionando como a camada de inteligência da cláusula WHERE.

1. LIKE (Busca de Padrões)

Diferente do operador de igualdade (=), o LIKE trabalha com aproximação e padrões de texto.

  • Cenário Básico: "Quero todos os nomes que começam com A". → LIKE 'A%'
  • Cenário Avançado: "Quero códigos de produto que seguem o padrão: duas letras, qualquer coisa no meio e terminam com o número 5". → LIKE 'AB%5'

2. BETWEEN (Faixa de Valores)

O grande diferencial do BETWEEN é a legibilidade e a limpeza do código.

3. IN (Membros de um Conjunto)

O IN atua como um "redutor de múltiplos operadores OR".

content_codigo: | 
-- [ MySQL ]
  SELECT descricao FROM produto WHERE descricao LIKE 'Gel%';
  
  -- [ PostgreSQL ]
  SELECT descricao FROM produto WHERE descricao ILIKE 'gel%';
content_grafico: |

content_dicas: |
  • Performance: Prefira LIKE 'Padrão%' a LIKE '%Padrão'.
  • Sensibilidade: PostgreSQL é case-sensitive (LIKE vs ILIKE).
  • Nulos: LIKE, IN e BETWEEN não capturam NULL.
content_pratica_real: |

E-commerce (Busca Inteligente): Implemente uma barra de pesquisa que utiliza LIKE '%termo%'.

Relatórios Financeiros: Utilize o BETWEEN para extrair todas as vendas ocorridas entre o primeiro e o último dia do mês.
